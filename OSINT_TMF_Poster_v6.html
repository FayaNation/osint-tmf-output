Introduction & Problem Statement

Background

Open Source Intelligence (OSINT) is the collection and analysis of publicly available information to support security operations. In cybersecurity, OSINT represents the reconnaissance phase of an attack — where adversaries gather intelligence before exploiting any vulnerability. Organisations unknowingly expose sensitive data through their websites, metadata, JavaScript endpoints, and subdomains.

The Problem

Public information is often collected without permission by attackers, but plenty of companies still don't use hands-on tools that show how OSINT threats work or why they are dangerous. Existing tools such as Maltego and SpiderFoot collect data well — but none simulate how that data translates into real attack paths.

This project addresses that gap by developing a platform that collects, analyses, and simulates real-world attack scenarios based on open data sources — bridging the gap between OSINT awareness and defensive security education.

Proposed Solution

A Five-Module Automated Framework

The solution is a five-module Python framework. A user provides a target URL and the entire pipeline runs automatically — from data collection through to a full threat report with MITRE-mapped attack scenarios.

1

Crawler Module

Passively fetches public pages and extracts emails, links, metadata, JavaScript endpoints, and subdomains. Respects robots.txt and applies rate limiting.

2

Enrichment Module

Structures raw data and enriches findings via Shodan (infrastructure intelligence) and HaveIBeenPwned (breach history) APIs.

3

Analyzer Module

Classifies each finding by type, assigns OWASP-inspired risk weights, detects PII using regex patterns, and computes a total risk score.

4

Simulator Module

Maps findings to MITRE ATT&CK tactics and generates evidence-based attack scenarios and multi-stage attack chains.

5

Reporter Module

Compiles findings into HTML, Markdown, and plain text reports with remediation recommendations.
